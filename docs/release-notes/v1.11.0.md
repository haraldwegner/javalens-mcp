# javalens-mcp v1.11.0 ‚Äî naming-convention + Javadoc knowledge tools (Sprint 15a)

Minor bump: **+2 tools (81 ‚Üí 83)**, both parametric, both read-only and **model-free**.
Gives autonomous agents two kinds of codebase knowledge they previously had to grep-and-guess:
how *this* codebase names things, and the API contracts already written in its Javadocs.

## analyze_javadocs(kind=‚Ä¶)

Read-only Javadoc knowledge, surfaced as deterministic facts ‚Äî never model-written prose.

- `ingest` ‚Äî parse existing Javadocs into symbol-anchored `symbol_fact`s
  `{ type, symbol, summary, details, source, confidence, evidence }`. Structured tags
  (`@param`/`@return`/`@throws`/`@deprecated`/`@see`) are machine-readable ‚Üí **HIGH**
  confidence (`api_contract`, `deprecated_behavior`); free-text prose is surfaced verbatim at
  **LOW** confidence (honest about what's deterministic ‚Äî no NLP contract extraction).
- `validate` ‚Äî doclint-style validation via the compiler (parser-scoped options, **no global
  side effect**): reports broken refs + missing tags on *documented* members. Deliberately
  does **not** enable missing-comment detection, so undocumented getters aren't spammed.
- `generate` ‚Äî emit a **doclint-correct Javadoc skeleton + extracted evidence** for a target
  symbol (FQN): `@param`/`@return`/`@throws` stubs, visibility/`@Deprecated`/declared-throws
  facts, and `prosePlaceholders`. **Model-free**: every slot is a `TODO` placeholder ‚Äî the
  calling agent writes the prose (the generation boundary; see
  [`docs/sprints/sprint-future-agent-runner.md`](../sprints/sprint-future-agent-runner.md)).
  Trivial getters/setters ‚Üí `skip:true`.

## analyze_naming(kind=‚Ä¶)

Read-only naming conventions, **shallow-precise**: name-pattern conformance + cheap
structural signals, confidence-gated, with recorded exceptions ‚Äî no deep body/role analysis.

- `infer` ‚Äî per-category conventions (type/method/field/constant/package/test) as
  `naming_convention` facts with confidence + examples + exceptions. A category below the
  min-sample threshold yields no convention (honest "unclear" instead of overfitting).
  Deterministic, sorted output.
- `get` ‚Äî the inferred conventions, optionally filtered by `category`.
- `suggest(category, intent)` ‚Äî applies the category's convention (casing + inferred test
  suffix) to the caller-supplied `intent`. **Model-free**: it only re-cases your words; with no
  `intent` it returns the convention *shape*, never an invented stem.
- `check(name, category)` ‚Äî validates a proposed name ‚Üí `conforms`, `violations`, and a
  re-cased `suggestion`.

## The shared symbol_fact schema

Both tools emit a single evidence shape (`org.javalens.mcp.knowledge.SymbolFact`):
`{ type, symbol | scope, summary, details, source, confidence, evidence, exceptions }`. This
is the contract the next javalens sprint (15b null-safety ‚Üí v1.12.0) and the later experience
store reuse, so knowledge derived here persists without reshaping.

## Design invariant: model-free

The server has no LLM. These tools emit facts, evidence, and doclint-correct skeletons ‚Äî
prose and semantic naming are the calling agent's job. That boundary is documented once in
[`sprint-future-agent-runner.md`](../sprints/sprint-future-agent-runner.md); `analyze_javadocs`
and `analyze_naming` depend on the handoff but introduce no model of their own.
