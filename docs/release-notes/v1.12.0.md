# javalens-mcp v1.12.0 — Java null-safety tools (Sprint 15b)

Minor bump: **+2 tools (83 → 85)**. Null-safety is the single biggest reason teams leave
Java for Kotlin, so this is the deepest slice of the modernization program — recovering much
of that safety inside the Java/JDT ecosystem, deterministically and with no LLM.

## analyze_nullness(kind=…) — read-only

- `detect_style` — identify the project's nullness annotation family (JSpecify / Eclipse JDT /
  JetBrains / JSR-305 / SpotBugs / Checker / AndroidX) by import scan. Works with no nullness
  jars on the classpath. Returns the dominant family + per-family counts + evidence.
- `find_violations` — enable JDT's null analysis via parser compiler options (parser-scoped,
  **no project mutation**) and report probable null bugs. Flow checks (null dereferences) work
  without annotations; annotation-contract checks activate when the project carries its
  nullness jar. Compiler-driven — not hand-rolled dataflow.
- `infer_contracts` — conservatively infer nullness contracts from strong signals
  (`Objects.requireNonNull` → `@NonNull`; explicit `return null` → `@Nullable`) as
  `null_contract` facts. **Emits NO contract inside framework-managed/generated types**
  (`@Generated`/`@Entity`/`@Component`/…) — a nullness annotation is an API contract, so false
  confidence is worse than none.
- `check` — focused: detected style + violations + inferred contracts for one file or symbol.

## apply_null_annotations(kind=…) — apply/undo

- `add` — insert `@Nullable`/`@NonNull` (+ import) on a method return, a parameter, or a field
  via the v1.9.0 apply/undo contract (`{ filesModified, diff, undoChangeId, summary }`;
  `auto_apply:false` to stage). Style defaults to the detected family, else **JSpecify**.
  **Public-API guard:** a public/protected target is refused unless `allowPublicApi:true` — it's
  a breaking downstream contract change.
- `migrate` — convert one nullness family to another (e.g. `JETBRAINS`→`JSPECIFY`): swaps the
  `@Nullable`/`@NonNull` usages + imports across a file or project. **Refuses ambiguous
  conversions** (default-scoping `@NonNullByDefault`/`@NullMarked`, wildcard imports,
  fully-qualified usages) rather than guess — family semantics are not 1:1.

## Two tools, by design

A single MCP tool can't both auto-carry `readOnlyHint` AND extend the mutating apply/undo
base, so null-safety ships as a read-only `analyze_nullness` (`analyze_` prefix →
`readOnlyHint:true`) plus a mutating `apply_null_annotations` — mirroring the
`find_modernization` / `apply_cleanup` split. Both emit the shared
`org.javalens.mcp.knowledge.SymbolFact` schema (a null contract is `type:"null_contract"`),
the same shape v1.11.0 introduced and the future experience store will persist. Model-free:
rationale prose is the calling agent's job ([generation boundary](../sprints/sprint-future-agent-runner.md)).

This completes the javalens content line; the next sprint is the GOJA rebrand (goja-mcp v1.0).
